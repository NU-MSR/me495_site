Zoom Setup

  1. In your group, one person is the leader.
    • The leader will share their screen and do most of the typing
    • If you can't decide on a leader, the person whose last name is last alphabetically is the leader
  2. Go through the steps in this activity as a group, while discussing answers to each question.
    • If there is some surprising behavior that you want explained more thoroughly note it in the Google doc under your Group number.
  3. The google doc is here. You must be logged in to your u.northwestern.edu account to see the google doc

Activity

  • Clone https://github.com/m-elwin/me495_urdf.git
  • See how the robot moves in rviz.
  • Change the launchfile so it no longer loads the rviz configuration.
    • Then, when rviz loads, set it up to see the robot.
  • Using the information in the URDF tutorial
    • Rename slidebot.urdf to slidebot.urdf.xacro and modify the launchfile to load the xacro file.
  • Create a parameter called base_radius and use it as the radius of the base sphere
  • Make the length of the arm 10 times the radius of the base sphere
  • Using the information here xacro YAML load the base_radius from a yaml file.
  • Use a xacro:macro to add a link to the arm that is the same as the current arm link but on the opposite side of the body

Today's Example

Here is what I did for today's example.

mkdir -p ws/src
cd ws/src
git clone https://github.com/jackal/jackal.git
git clone https://github.com/ros-drivers/pointgrey_camera_driver.git
cd ws
catkin_make
xacro src/jackal/jackal_description/urdf/jackal.urdf.xacro > jackal.urdf
roslaunch urdf_tutorial display.launch model:=jackal.urdf
