Homework 4

Overview

The same rules for turning in this assignment apply as for the previous assignments.

  1. Don't forget to commit to git often.
  2. Include a README.md that describes the nodes and launchfiles in your package.
    • Make sure your README.md is formatted correctly
  3. Remember to include docstrings in each node describing its ROS API

Image Processing

  1. Follow the instructions to obtain YOLO ROS, a package that enables real-time object detection in ROS: https://github.com/leggedrobotics/darknet_ros

  2. Write a launchfile and rviz config files that sets up your webcamera and YOLO ROS to detect objects and view the results in RViz in real time.

    • Include a screenshot of you using this in your README.md
    • Downloading and compiling this package may take a while! Do something else while it compiles.

Navigation

The turtlebot3 tutorials (especially those related to simulation) will be a useful reference for you

  1. Write a launchfile that loads the turtlebot3 in the house environment, loads rviz and visualizes it's laser scan, starts a teleoperation node, and gmapping.

    • You will need to set the TURTLEBOT3_MODEL environment variable to burger to use the launchfile
    • Many of these actions can be accomplished by including appropriate launchfiles from turtlebot3 packages. However, if you want more control you can use these launchfiles as inspiration rather than directly including them
    • You should make sure that rviz shows the Map, the laser scan, and the robot.
    • The fixed frame in rviz should be the map frame.

  2. Drive the turtlebot3 throughout the interior of the house (at least two rooms) while running gmapping. Use the mapserver to save the map file. Include both the resulting .yaml file and .pgm file in your git repository.

  3. Create launchfile that uses amcl to localize your robot, using the map created in the previous step.

    • You should also launch the move_base so that you can use 2D navigation goals in rviz to move your robot.
    • Use parameters to pass it a reasonable initial pose.
    • Try to make the global or local planner fail
    • Create a screenshot showing the plan being executed in RViz.
      • The robot, the motion plan, the static map, the costmap, and the costs for the global plan should be visible.
      • You should show the Path that is planned

  4. After running the setup above, add some new obstacles into the gazebo world.

    • Some obstacles should be added within sight of the laser scanner and can be immediately incorporated into the global plan
    • Other obstacles that are not immediately visible will need to be avoided with local planning
    • Send goals that require the robot to avoid the obstacles.
    • Include a screenshot in your README.md of the turtlebot moving around an obstacle in rviz

  5. Write a node that causes the robot to explore the environment autonomously while mapping the world with gmapping.

    • Provide a launchfile that lets the robot use this node and gmapping to autonomously map the environment
    • You can use any algorithm you want: if you use an existing algorithm cite it
    • You can subscribe to the /map topic to get information about the robot's current knowledge of the environment
    • Your algorithm can issue cmd_vel commands directly or work in conjunction with the move_base node to plan via waypoints.
    • Include a brief description of your strategy in the README.md
    • Include a map that you created after running the exploration algorithm in your repository and refer to it in your README.
    • The exploration does not need to be complete, perfect, or optimal in any way.
    • You may wish to test your algorithm at multiple initial poses.
    • There are purposely no performance constraints on this task; however, you should come up with a strategy, describe that strategy in the README.md, and make sure that you have actually implemented what you intended to implement.

Hints
