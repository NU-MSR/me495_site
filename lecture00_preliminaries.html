Installing ROS

Here are instructions for installing ROS noetic and some packages that have not yet been released for noetic (which only came out in May 2020). Since we are compiling some packages from scratch, you can expect this total process to take approximately 1 hour.

  1. Install ROS Noetic (Choose the desktop-full version and follow all of the instructions)

  2. Install python3-rosdep. Make sure you have installed ROS first or this will cause many problems down the road.

    sudo apt install python3-rosdep
sudo rosdep init
rosdep update
    • Don't forget to echo "source /opt/ros/noetic/setup.bash" >> ~/.bashrc
    • Caution: if you try to run rosdep prior to installing it, the similarly-named python3-rosdep2 will be suggested. Do NOT install it.

  3. Install some dependencies that we need to build everything: sudo apt install python3-vcstool git gitk python3-pip.

  4. Install some dependencies that are not properly accounted for by the package's ROS dependency system

    sudo apt install ros-noetic-dynamixel-sdk libnlopt-cxx-dev
pip3 install modern_robotics

  5. Install librealsense2, as compiled by me (see below for how if you are curious):

    curl -L "http://drive.google.com/uc?export=download&id=1zPS8Q84a59-GDaZKQuR4KwtUS3zzoc56" \ 
> /tmp/librealsense.deb
sudo apt install /tmp/librealsense.deb

  6. We will now create a ROS workspace that contains packages that have not yet been released for noetic. This step takes a long time so be patient!

    source /opt/ros/noetic/setup.bash
mkdir -p ~/nuws/src
cd ~/nuws/src
vcs import --input https://nu-msr.github.io/me495_site/nuws.repos
cd ~/nuws
rosdep install --from-paths src --ignore-src -r -y
catkin_make -DCMAKE_BUILD_TYPE=RelWithDebInfo
    • You may see an error about orocos_kdl and librealsense2 when running rosdep. These are expected.

  7. Install the udev rules for the pincherX 100

    • sudo cp src/interbotix_ros_arms/interbotix_sdk/10-interbotix-udev.rules /etc/udev/rules.d

  8. To use the packages you just compiled you must source the workspace.

    • source ~/nuws/devel/setup.bash
    • Append to your .bashrc: echo "source ~/nuws/devel_isolated/setup.bash" >> ~/.bashrc

Building librealsense

For reference, here are the steps I used to build librealsense I also use checkinstall as a quick hack for creating .deb packages from the results. I do not recommend this approach: checkinstall can completely destroy your system. You need not follow any of these steps if you downloaded the binary provided by me. Compile the library

git clone https://github.com/IntelRealSense/librealsense.git
cd librealsense
git checkout v2.38.1
cmake -B build -DCMAKE_BUILD_TYPE=RelWithDebInfo -DCMAKE_INSTALL_PREFIX=/usr -DBUILD_PYTHON_BINDINGS:bool=true .
cd build
make -j8
  • There is a race condition in build paraellization so if you run into an error just run make again.

Create the installation script, called makeit and place it in build/

#!/usr/bin/env bash
cd ..
./scripts/setup_udev_rules.sh
cd build
make install
mv /usr/lib/python3/dist-packages/pyrealsense2/*.so /usr/lib/python3/dist-packages
mv /usr/lib/python3/dist-packages/pyrealsense2/*.so.* /usr/lib/python3/dist-packages

Make the script executable: chmod 755 makeit

Run checkinstall to create the package sudo checkinstall --pkgname=northwestern-librealsense --provides=ros-noetic-librealsense2 --conflicts=ros-noetic-librealsense2 ./makeit

  • Exclude files from the home directory from the package
  • You can edit metadata if you got it wrong on the initial checkinstall by following this guide
